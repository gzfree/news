Botim Search AI Enhancement Features

🪟 概述

本项目旨在通过引入以下功能来增强Botim的搜索功能：

  • 网页搜索集成：当本地搜索结果为空时，提供网页搜索结果作为替代。
  • 多语言搜索支持：支持阿拉伯语和中文内容的搜索，同时保持相关性匹配。
  • 内部知识库集成：实现固定规则和基于LLM的语义匹配的双层匹配系统。
  • IM消息搜索：为IM对话中的选定文本消息添加搜索功能。
  • 金融服务集成：扩大搜索范围以包含更多金融服务。
  • 社交证明：显示每个服务的使用统计（例如"1000+用户"）。
  • 服务标签：为服务添加描述性标签，以便更好地在UI中展示和说明功能。

🛸 问题

  • 当本地搜索结果不足时，用户需要访问网页内容。
  • 语言障碍限制了非英语内容的搜索效果。
  • 内部知识库内容未在搜索结果中充分利用。
  • 用户无法轻松搜索IM对话内容。
  • 金融服务无法通过搜索完全发现。
  • 服务可信度和受欢迎程度信息对用户不可见。
  • 服务功能未通过标签清晰传达。

🎯 目标

  • 无缝集成网页搜索作为本地结果的补充
  • 优化阿拉伯语和中文内容的搜索体验
  • 构建高效的混合搜索系统（规则+LLM）
  • 实现IM对话内文本搜索功能
  • 增强金融服务的可发现性
  • 通过使用统计增强服务可信度
  • 使用标签提升服务分类与描述

🚧 限制条件

  • 网页搜索结果必须与本地结果明确区分。
  • 多语言支持必须保持高准确性和相关性。
  • 基于LLM的匹配必须优化性能和成本。
  • 消息搜索必须遵守用户隐私和数据保护法规。
  • 服务标签必须在整个平台上标准化和一致。

📌 场景2：用户从IM消息中选择文本并搜索相关服务或信息。

📌 场景3：当本地搜索没有结果时，系统自动显示相关的网页搜索结果。
